# 変更履歴

[English](CHANGELOG.md) | 日本語

pocket-claude の主要な変更をここに記録します。

## [未リリース] - 2026-03-28

### 追加
- **予約投稿** - 指定日時にプロンプトを自動送信；出力パネルから編集・キャンセル可能；`scheduled-posts.json` に永続化（サーバー再起動後も復元）
- **言語切り替え** - 設定画面から日本語 / 英語 を切り替え可能；localStorage に永続化
- **systemd 検知バッジ** - systemd 管理外で起動した場合にヘッダーに警告を表示；`/api/health` エンドポイントが `systemd_managed` フラグを返す
- **レート制限自動再開** - レート制限発生時にパネルを表示してサーバーサイド自動再開を有効化；スケジュールを `schedules.json` に永続化
- 新規タブ作成時に現在のタブのモデル・Effort・Thinking 設定を継承

### 変更
- 設定 UI の Effort・Thinking ラベルを日本語化
- モデルセレクタのデフォルトラベルをアルファベット `default` 固定（レイアウトずれ防止）
- ヘッダーボタンの高さを 30px に統一
- 予約ボタンを SVG アイコンに変更（絵文字は高さが揃わないため）

### 修正
- プロジェクト切り替え後、設定を閉じずに選択状態を即時更新するよう修正
- `parseResetTime` が `am/pm + IANA タイムゾーン` 形式に対応
- 予約リスト表示の競合修正：`loadHistoryBulk` 完了後にのみ `loadScheduledPosts()` を呼ぶよう変更

## [v4.35] - 2026-03-23

### 修正
- CLI コマンドのパースエラー（`type: "result"`、`is_error: true`）が無視されていた問題を修正；赤字で表示するよう対応

## [v4.34] - 2026-03-22

### 変更
- モデルセレクタのテキストを左揃えに変更（Safari `text-align-last` 対応）
- Send ボタンの高さが textarea の高さに連動するよう変更

## [v4.32–v4.33] - 2026-03-22

### 変更
- ツール展開表示を VS Code 拡張スタイルに刷新：Bash は IN/OUT 形式、Edit は赤/緑の diff 形式
- Grep の説明形式を `"pattern" (in /path)` に統一
- 通信断ステータスドットの外枠を削除・赤色に変更
- ツールパラメーター表示が出力フォントサイズを継承するよう変更

## [v4.31] - 2026-03-22

### 追加
- コードフォントサイズ設定を追加（デフォルト 12px、`--code-font-size` CSS 変数で制御）

### 修正
- ask-text・ask-option・tool-result 等のフォントサイズがハードコードされていて出力フォントサイズを継承していなかった問題を修正

## [v4.30] - 2026-03-22

### 追加
- レート制限自動再開：レート制限が解除されるタイミングでサーバーサイドがプロンプトを自動再送信；UI にカウントダウン表示；リセット時刻を Claude のメッセージから自動抽出

## [v4.28–v4.29] - 2026-03-21

### 追加
- Thinking コントロールをチェックボックスから 3 ステートセグメント `指定なし | オン | オフ` に変更

### 修正
- レート制限メッセージを `assistant` イベント（`ev.error === "rate_limit"`）から検出して赤字表示するよう修正

## [v4.27] - 2026-03-21

### 追加
- AskUserQuestion ツール結果を自動展開して質問・選択肢をリスト表示

### 修正
- `--include-partial-messages` が `input_json_delta` を破損させ JSON.parse が失敗していた問題を修正；完全な `assistant` メッセージデータを使って描画するよう変更
- HTTP 環境向け `crypto.randomUUID` ポリフィルを追加
- ルートレスポンスに `Cache-Control: no-store` を追加

## [v4.26] - 2026-03-21

### 変更
- stderr の使用制限メッセージを `[err]` プレフィックスなしで `line-error`（赤・通常サイズ）で直接表示
- リセット時刻を目立つ表示に変更
